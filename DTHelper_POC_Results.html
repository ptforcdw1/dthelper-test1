Objectives

What this POC set out to prove

Automate DT Configuration
Perform Settings API v2 operations through natural language — no manual Dynatrace UI required.
Natural Language Interface
Users describe tasks in plain English; Claude handles schema selection, payload construction, and API calls.
Full Auditability
Every task gets a unique ID (TASK-YYYYMMDD-NNN), committed to Git, and logged in activity_log.md.
Jenkins-Driven Execution
All API calls run inside Jenkins pipelines — no direct execution from the chat session.
Reusable Templates
Common operations are extracted into parameterized templates to eliminate repetitive script writing.
No DT Expertise Required
Operators can complete complex configuration tasks with zero knowledge of Dynatrace schema names or API paths.

Design Approach

Key architectural decisions and why they were made

Orchestrator Pattern
Claude never calls APIs directly. It generates fully parameterized Python scripts committed to Git, then triggers Jenkins. This keeps a complete audit trail and separates intent from execution.
Template Registry
Reusable operations (metric alerts, log routing) are extracted into templates with a guideline.md. New tasks use templates whenever possible — cutting implementation from 9 Jenkins builds down to 1.
7-Step Workflow
Every task follows: Clarify → Check Templates → Generate ID → Commit/Push → Create Pipeline → Trigger Build → Update Log. Steps are deterministic and fully auditable.
Separation of Concerns
environment.json is the single source of truth for URLs and config. Secrets stay in the Jenkins credentials store, never in code. Scripts are stateless — all inputs via argparse.

Mock-First Development: All tasks were validated against httpbin.org (an echo endpoint) before any real Dynatrace tenant was configured. Scripts detect mock mode automatically and adjust test pass logic accordingly.

Claude Code Skills

Custom slash commands that enforce consistent behaviour across every session

Skills are project-level prompt files stored in .claude/commands/. When invoked with /skill-name, they inject a structured instruction set into the conversation — ensuring Claude follows the correct workflow every time, even in a fresh session with no prior context.

/task <request>
.claude/commands/task.md

Orchestrates the full 7-step task workflow in strict order — no steps can be skipped.

  1. Clarify requirements & resolve missing inputs
  2. Check template_records.md for a matching template
  3. Generate the next sequential task ID
  4. Write execute.py, test.py, metadata.json → commit & push
  5. Generate Jenkinsfile → register pipeline via REST API
  6. Report to user — wait for confirmation before triggering
  7. Append entry to activity_log.md
Example
/task create alert when error rate exceeds 5% for 3 minutes
/status
.claude/commands/status.md

Reads project state and returns an instant orient-me snapshot — useful at the start of every session.

  • Last 5 task entries from activity_log.md (ID, description, result)
  • All available templates from template_records.md (name, use case, approval)
  • What the next task ID will be for today
  • One-line summary — any failures or pending items flagged

# tasks/task_TASK-20260510-001/execute.py

def build_payload(args):
    samples = args.duration_minutes
    return {
        "schemaId": "builtin:anomaly-detection.metric-events",
        "scope": "environment",
        "value": {
            "enabled": True,
            "summary": args.summary,
            "queryDefinition": {
                "type": "METRIC_SELECTOR",
                "metricSelector": "builtin:host.cpu.usage:avg:auto"
            },
            "modelProperties": {
                "type": "STATIC_THRESHOLD",
                "threshold": args.threshold,           # 90
                "alertCondition": "ABOVE",
                "violatingSamples": samples,         # 10  (= 10 minutes)
                "samples": samples,
                "dealertingSamples": max(1, samples // 2)
            },
            "eventTemplate": {
                "title": "High CPU Usage on Host",
                "eventType": "PERFORMANCE",
                "davisMerge": True
            },
            "alertingScope": []                    # all hosts
        }
    }

B2 — Jenkinsfile: Task Pipeline (TASK-20260510-001)

jenkins/pipelines/task_TASK-20260510-001/Jenkinsfile  |  Declarative pipeline — Checkout → Validate → Execute → Test

// jenkins/pipelines/task_TASK-20260510-001/Jenkinsfile

pipeline {
  agent any
  parameters { booleanParam(name: 'DRY_RUN', defaultValue: false) }
  environment {
    DT_API_TOKEN = credentials('DT_API_TOKEN')   // Jenkins credentials store
    TASK_DIR     = 'tasks/task_TASK-20260510-001'
    BRANCH       = 'master'
  }
  stages {
    stage('Checkout') { steps {
      git branch: env.BRANCH, url: 'https://github.com/ptforcdw1/dthelper-test1.git'
    } }
    stage('Validate') { steps {
      sh 'python3 --version'
      sh 'pip3 install requests --quiet --break-system-packages'
    } }
    stage('Execute') { steps {
      sh """python3 ${TASK_DIR}/execute.py \
          --summary 'High CPU Usage Alert' \
          --metric-selector 'builtin:host.cpu.usage:avg:auto' \
          --threshold 90 --duration-minutes 10"""
    } }
    stage('Test') { steps {
      sh "python3 ${TASK_DIR}/test.py --threshold 90"
    } }
  }
}

B3 — execute.py: Log Routing Rule (TASK-20260510-003)

tasks/task_TASK-20260510-003/execute.py  |  Schema: builtin:logmonitoring.log-storage-settings  |  Routes k8s namespace logs to Grail bucket

# tasks/task_TASK-20260510-003/execute.py

def build_payload(args):
    rule_name = args.rule_name or \
        f"Route {args.matcher_attribute}={args.matcher_value} to {args.bucket_name}"
    return {
        "schemaId": "builtin:logmonitoring.log-storage-settings",
        "scope": "environment",
        "value": {
            "enabled": True,
            "config-item-title": rule_name,
            "matchers": [{
                "attribute": args.matcher_attribute,  # k8s.namespace.name
                "operator":  args.matcher_operator,   # EQUALS
                "values":    [args.matcher_value],     # test-ns
            }],
            "bucketName": args.bucket_name,           # testapp-bucket
        }
    }

# Called with:
# python3 execute.py --bucket-name testapp-bucket \
#   --matcher-attribute k8s.namespace.name --matcher-value test-ns

# templates/create-metric-alert/execute.py

parser = argparse.ArgumentParser()
parser.add_argument('--summary',          required=True,  help='Alert display name')
parser.add_argument('--metric-selector',  required=True,  help='DT metric selector')
parser.add_argument('--threshold',        required=True,  type=float)
parser.add_argument('--alert-condition',  default='ABOVE', choices=['ABOVE','BELOW'])
parser.add_argument('--duration-minutes', default=5,      type=int)
parser.add_argument('--event-type',       default='PERFORMANCE')
parser.add_argument('--dry-run',          action='store_true')
